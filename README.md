# mqtt-monitor

> MQTT 기반 리눅스 서버 실시간 모니터링 + 원격 관리 대시보드

---

## 개요

**mqtt-monitor**는 단순한 시스템 모니터링을 넘어, 헤드리스 미니PC를 SSH 없이 브라우저에서 직접 관리하기 위해 만든 프로젝트입니다.

모니터·키보드 없이 24시간 가동 중인 서버를 Tailscale로 원격 접속해 운영하다가, 반복적인 SSH 접속이 번거로워 제작했습니다.

- systemd 서비스를 브라우저에서 직접 시작/중지/재시작
- Tailscale 연결 기기 온/오프라인 상태 확인
- 시스템 재시작·종료 (텍스트 입력 확인 모달로 오조작 방지)
- CPU·메모리·디스크·온도 임계값 초과 시 Discord 알림

## 주요 기능

### 모니터링 탭
- 실시간 게이지: CPU 사용률, CPU 온도, 메모리, 디스크, 네트워크 속도
- 최근 10분 시계열 차트
- 네트워크 인터페이스 자동 감지
- CPU 온도 센서 자동 감지 (`coretemp` → `k10temp` → `acpitz`)

### 관리 탭
- **서비스 제어** — 설정된 systemd 서비스 시작 / 중지 / 재시작
- **Tailscale 기기 현황** — tailnet 내 기기별 온/오프라인 상태
- **시스템 제어** — 재시작·종료 (텍스트 직접 입력 모달로 오조작 방지)

### 알림
- 임계값 초과 시 Discord 웹훅 알림
- 10분 쿨다운으로 스팸 방지

| 메트릭 | 임계값 |
|---|---|
| CPU 사용률 | 85% 초과 |
| CPU 온도 | 80°C 초과 |
| 메모리 | 90% 초과 |
| 디스크 | 90% 초과 |

## 기술 스택

| 역할 | 기술 |
|---|---|
| 메트릭 수집 | `psutil` |
| 메시지 브로커 | Mosquitto (MQTT) |
| 데이터 저장 | SQLite |
| 백엔드 | Flask |
| 프론트엔드 | Vanilla JS + Chart.js |
| 알림 | Discord Webhooks |
| 프로세스 관리 | systemd / Docker |

---

## MQTT를 사용한 이유

수집(publisher) / 저장(subscriber) / 시각화(dashboard) 세 컴포넌트를 MQTT 브로커를 통해 완전히 분리했습니다.

모니터링 대상 머신을 추가할 때 `publisher.py`만 새 머신에 배포하면 되고, subscriber와 dashboard는 수정 없이 확장됩니다. 같은 브로커에 여러 subscriber를 붙이면 DB 저장과 알림을 독립적으로 운용하는 것도 가능합니다.
